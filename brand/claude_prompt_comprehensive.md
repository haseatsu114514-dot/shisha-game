# Claude Code への統合指示プロンプト（ストーリー＋技術ドキュメント構成）

以下のテキストをコピーして、Claude Code（Claude 4.6等）のチャットに貼り付けてください。

---

あなたはプロのシナリオライター、ゲームプランナー、およびテクニカルアーキテクトです。
現在開発中である「シーシャ屋を舞台にしたアドベンチャー＆シミュレーションゲーム」のプロジェクトについて、以下の【1. 全キャラクター設定の再添削とデータ化】および【2. 今後のMD（技術設計ドキュメント）構成の提案】の両面から支援をお願いします。

## プロジェクトの背景情報を読み込んでください
まず、プロジェクトルートにあるリポジトリの `.claudemd` を読み、次に以下の2つの公式資料を確認して、全体像を把握してください。
- `brand/character_profiles.md`
- `brand/story_and_structure.md`

## 依頼事項

### 1. 全キャラクター設定の再添削とデータフォーマット化
主人公「はじめ」は、相手を否定せず目の前の客だけを見る「フラットな優しさ」が武器です。
このテーマを軸とし、彼と対峙する**すべてのキャラクター（なる、アダム、眠都、ナンディ、SHISHA-9000、スティーブ、ヴォルク、マスター・フーカなど）**について設定の深掘りと再解釈を行ってください。

- **A. 設定のブラッシュアップ**: 各キャラクターの「シーシャの作り方・こだわり（得意フレーバーや熱管理）」が、彼ら自身の「生き方の不器用さ」「過去のコンプレックス」「秘めたる想い」とどう直結しているか、より深みのある裏設定として添削・追記してください。
- **B. 既存データ（JSON）のブラッシュアップと上書き**: プロジェクトの `data/characters.json` に、初期設定のプロフィールがリスト構造でまとまっています。あなたが考えた深みのある裏設定やプレイスタイルの再解釈を、この JSON の `background_secret` や `description` の項目に統合し、ファイルを上書き出力してください。
- **C. 成長・交流ステータスの定義**: ゲーム内で主人公が各キャラクターと交流（お出かけ等）した際、「誰についていけば、どのステータス（技術、センス、根性、魅力、洞察など）が伸びやすいか」という方針をキャラクターごとに考案してください。考案した内容は、JSONデータの `growth_stats` というプロパティに追記して保存してください。主人公が目指したい方向性（ステータスビルド）に直結する重要なゲームバランス設計となります。
- **D. キャラクタービジュアル案の作成**: 各キャラクターにつき、外見・服装・雰囲気などのデザイン案（ビジュアルコンセプト）を **だいたい2案ずつ** 考案してください。考案した案は、JSONデータの `visual_concepts` （配列要素）として追記してください。


### 2. 今後の技術設計ドキュメント（MDファイル）構成の策定
現在、プロジェクト全体の仕様やシステム設計をMarkdownファイル群として整理し直そうとしています。
Godot 4を用いたこのアドベンチャー＆シミュレーションゲームを今後拡張していくにあたり、保守性を高く保ちつつ、AI（あなたや他のコーディングエージェント）が継続的に参照しやすいような「技術設計ドキュメント群（ディレクトリ構造含む）」の全体構成案を提示してください。

想定される要素は以下の通りです：
- データ構造の定義（キャラクターフラグ、好感度、アイテム、シーシャ作成パラメータなど）
- システムアーキテクチャ（GameManagerなどのAutoload、UIフロー、画面遷移）
- シナリオフォーマット（現状のJSON形式の実装仕様、追加イベントの手順）
- その他、開発・運用上必要と思われるルールや技術ドキュメント

どのようなディレクトリ/ファイル名（例：`docs/architecture.md`）で、各ファイルにどのような項目（目次）を記載すべきか、ベストプラクティスに基づいた構成ツリーを出力してください。
