## 前提資料（必ず全て読んでから作業してください）

2. **`brand/story_and_structure.md`** — 全4章のストーリー構成とテーマ
3. **`data/characters.json`** — 現在のキャラクターデータ（JSON形式）
4. **`data/dialogue/*.json`** — 各キャラクターの実装済み会話データ（口調の把握に使用）

## ゲームのコアコンセプト（最重要）

### 主人公の強み：「フラットさ」
主人公の強みは「フラット」であること——**誰のスタイルも否定しない、違いをそのまま受け入れる力**です。
（※旧設定にあった「客観視」や「客を見る力」という設定は廃止されています。使わないこと。）

主人公は特別な才能を持っていません。だからこそ、関わった人から自然と何かを吸収し、「気づいたら変わっていた」という形で成長していきます。

### ステータスの成長
各キャラクターにはそれぞれ深いテーマがあり、プレイヤーが交流する中でステータスが上がります。
- これは「教わる」のではなく、**「この人と関わっていく中で、勉強になった部分が自然と身についていた」**という感覚が大切です
- `growth_stats.notes` は「○○から学んだ」ではなく **「いつの間にか△△が変わっていた」** という視点で書くこと

### 夢シーンシステム
- 夜→朝の遷移時（就寝→起床）に、**1章につき1〜2回**、主人公が夢を見るシーンがある
- 今後の章の伏線、スミさんの過去の断片、ライバルの言葉が煙の中に浮かぶ抽象的な夢、隠しボスへの予兆など
- テキストは詩的・断片的に。プレイヤーが後から「あれはこのことだったのか」と気づく構造にすること

## 依頼内容

### 1. 設定・ストーリーの添削

`brand/story_and_structure.md` および `brand/character_profiles.md` を参照し、以下を実施してください：

- **各キャラクターの「生き様」とシーシャ製法のリンク強化**
  どのキャラクターも「なぜ今このやり方でシーシャと向き合っているのか」という動機が、その人の人生と直結するように添削してください
- **第2章ライバル陣（組長・レイ・ドクター・ケムリ・アゲハ）の掘り下げ**
- **各章の「夢シーン」で何を暗示するかの提案**（1章〜4章それぞれ1〜2個ずつ）
- **矛盾点・穴の指摘**

### 2. `data/characters.json` の更新（上書き出力）

各キャラクターに以下のフィールドを追加・充実させて、ファイルを上書き出力してください：

```json
{
  "background_secret": "（添削した裏設定・バックボーン）",
  "growth_stats": {
    "primary": "（交流で最も伸びるステータス: 技術 / センス / 根性 / 魅力 / 洞察）",
    "secondary": "（サブで伸びるステータス）",
    "notes": "（このキャラと関わってどう変わるか。「いつの間にか○○が変わっていた」という視点で書くこと）"
  },
  "visual_concepts": [
    "（外見コンセプト案①：イラストレーターに口頭説明できるレベルで具体的に）",
    "（外見コンセプト案②）"
  ]
}
```

**重要な個別設定：**
- **主人公（はじめ）** — 強みは「フラット」のみ。「客を見る」「客観視」は旧設定なので使用禁止
- **眠都（みんと）** — `background_secret` に「公称20歳だが実際はアラサーで20代後半。本人は完全に隠している」を必ず含める
- **かこ** — シーシャを「作ること」が好き。好きフレーバーは Bombshell と White Musk。VTuber・ネットミーム好きの「ホワイトオタク」
- **南雲修二** — 大手チェーン「シーステーション」会長兼大会審査員長、45歳

### 3. 技術ドキュメント構成の提案

`docs/` ディレクトリに配置する開発用 Markdown 資料の目次ツリーを提案してください。

## 出力形式

- JSON → `data/characters.json` へ直接上書き
- 添削コメント・夢シーン提案・docs構成案 → チャット上に出力

## トーン

- シーシャ特有の「熱管理・煙の質感・フレーバー製法」などを比喩に使い、専門的かつ情緒的なテキストを心がける
- ビジュアルコンセプトは「イラストレーターに口頭説明できる」具体性（髪型・服装・色・雰囲気など）
