# Claude Code 向け 統合作業プロンプト

> このファイルの内容を Claude Code のチャットに貼り付けてください。

---

あなたはプロのシナリオライター兼ゲームプランナーです。以下の指示に従い、`shisha-game` プロジェクトのデータ整備を行ってください。

---

## 前提資料（必ず読んでから作業してください）

作業前に以下のファイルを必ず読み込んでください：

1. **`brand/character_profiles.md`** — 全キャラクターのプロフィール（章・役割・スタイル・年齢）
2. **`brand/story_and_structure.md`** — 全4章のストーリー構成とテーマ
3. **`data/characters.json`** — 現在のキャラクターデータ（JSON形式）
4. **`data/dialogue/*.json`** — 各キャラクターの実装済み会話データ（口調の把握に使用）

---

## 依頼内容

### 1. 設定・ストーリーの添削

`brand/story_and_structure.md` および `brand/character_profiles.md` を参照し、以下を実施してください：

- **各キャラクターの「生き様」とシーシャ製法のリンク強化**  
  例：スミさんの「基本に忠実」= 過去の放浪で学んだ「どの場所でも通じる普遍性」であること、など
- **第2章ライバル陣（組長・レイ・ドクター・ケムリ・アゲハ）の掘り下げ**  
  各キャラが「なぜ今シーシャを作っているのか」という動機を、それぞれの人生背景と絡めて添削してください
- **矛盾点・穴の指摘**  
  設定間の矛盾や「ここをもう少し詰めると良い」という箇所を指摘してください

### 2. `data/characters.json` の更新（上書き出力）

`data/characters.json` を読み込み、以下のフィールドを各キャラクターに追加・充実させて上書き出力してください：

```json
{
  "background_secret": "（添削した裏設定をここに追記）",
  "growth_stats": {
    "primary": "（主人公が交流で最も伸びるステータス名: 技術 / センス / 根性 / 魅力 / 洞察）",
    "secondary": "（サブで伸びるステータス名）",
    "notes": "（なぜそのキャラと付き合うとそのステータスが伸びるのかの理由）"
  },
  "visual_concepts": [
    "（外見・服装・雰囲気のビジュアルコンセプト案①）",
    "（外見・服装・雰囲気のビジュアルコンセプト案②）"
  ]
}
```

**※ 眠都（みんと）の `background_secret` については「公称22歳だが実際はかなり高齢という設定がある。本人は完全に隠している。」という内容を必ず含めること。**

### 3. 技術ドキュメント構成の提案

現在のプロジェクトに `docs/` ディレクトリを設け、以下のような開発用 Markdown 資料の目次ツリーを提案してください：

```
docs/
├── system_design/    （ゲームシステムの設計書）
├── scenario/         （シナリオ・イベント設計）
├── data_format/      （JSONデータの定義書）
└── ui/               （画面遷移・UI仕様）
```

各カテゴリに「どんなファイルが必要か」を具体的に列挙してください。

---

## 出力形式

- JSON の上書きは `data/characters.json` ファイルへ直接書き込んでください
- 技術ドキュメントの提案は Markdown 形式でチャット上に出力してください
- 添削コメントはチャット上に出力してください（既存ファイルの書き換えは不要）

---

## トーン・注意事項

- シーシャ特有の「熱管理・煙の質感・フレーバー製法」などを比喩に使い、情緒的かつ専門的なテキストを心がけてください
- ビジュアルコンセプトは「イラストレーターに口頭で説明できる」レベルの具体性を持たせてください
- 設定に矛盾が見つかった場合は、その場で修正案を提示してください
